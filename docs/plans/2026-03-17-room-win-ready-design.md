# 房间区分、胜利展示、自动准备流程 — 设计文档

**日期：** 2026-03-17
**状态：** 已确认

---

## 1. 概述

完善德州扑克游戏的三个交互流程：
1. 房间创建/加入的视觉区分
2. 每局结束的胜利标识
3. 准备系统实现

---

## 2. 房间区分

### 2.1 Lobby 视觉区分

- **创建房间模式**：绿色主题（`#27ae60` 按钮）
- **加入房间模式**：蓝色主题（`#3498db` 按钮）

### 2.2 房间号展示

- 创建者进入房间后，房间号以醒目方式展示，便于分享给其他玩家

---

## 3. 胜利展示

### 3.1 显示时机

- 游戏进入 SHOWDOWN 阶段后
- 胜者确定时（1 秒后自动消失）

### 3.2 显示内容

- 胜者名字（大字）
- 赢得筹码数量
- 居中显示于牌桌中央

### 3.3 样式

- 金色/黄色高亮文字
- 1 秒后自动消失，切换到 WAITING 状态

---

## 4. 准备系统

### 4.1 状态

- 每局结束后，玩家状态从 `waiting` 变为 `ready`
- 房主始终可以看到"开始游戏"按钮

### 4.2 玩家操作

- **非房主**：显示"准备"按钮，点击后状态变为 `ready`
- **房主**：显示"开始游戏"按钮，点击后开始下一局

### 4.3 逻辑

```javascript
// 玩家状态
player.status: 'waiting' | 'ready' | 'active' | 'folded' | 'allin' | 'out'

// 游戏结束后
phase = 'WAITING'
players.forEach(p => {
  if (p.chips > 0) p.status = 'ready'  // 新状态
})

// 房主点击开始
room.startGame()  // 需要所有玩家都 ready？
```

---

## 5. 后端改动

### 5.1 新增玩家状态

`ready` - 准备状态，表示玩家已准备下一局

### 5.2 事件

- `player:ready` - 玩家点击准备
- `player:unready` - 玩家取消准备

### 5.3 开始条件

房主可以随时点击开始，或设置所有玩家都 ready 后才能开始（可选）

---

## 6. 前端改动

### 6.1 Lobby.jsx

- 创建房间按钮：绿色主题
- 加入房间按钮：蓝色主题

### 6.2 Table.jsx

- WAITING 阶段根据玩家身份显示不同按钮：
  - 房主："开始游戏"
  - 其他玩家："准备/已准备"
- SHOWDOWN 阶段：显示胜利信息（1秒）

### 6.3 useGame.js

- 新增 `doReady()` 方法

---

## 7. 交互流程

```
WAITING → PREFLOP → ... → SHOWDOWN → (显示胜者1秒) → WAITING(ready状态)
                                                          ↓
                                              房主点击"开始游戏"
                                                          ↓
                                               PREFLOP → ...
```
